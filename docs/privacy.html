Privacy Policy

Where Was I — browser extension

Last updated: October 26, 2025

Overview

This Privacy Policy explains what data the extension may collect, how it is stored and used, and how you can control it. The extension is designed to store session information locally by default and only sends data off-device when you explicitly enable an external API or server.

Data Collected

Depending on your settings the extension may collect:

  • Tab metadata: page title, URL, favicon, and timestamps when tabs are saved or closed.
  • Optional scraped content: a short text snippet or summary extracted from a page when you enable the scraper feature.
  • Settings and preferences: feature flags (for example trackAllSites), UI preferences, and any API key you choose to provide for third-party services.

How Data Is Used & Stored

All data is stored locally using the browser's extension storage (typically chrome.storage.local). Data remains on your device unless you explicitly opt into features that send data to a third-party AI provider or enable a server proxy.

Third-Party Services

AI integrations and external servers are optional. If you configure an API key or enable the provided server proxy, the extension may send scraped content or session metadata to that service to generate summaries or titles. You are responsible for reviewing the third-party provider's privacy practices. The default extension behavior does not send data to remote servers.

Permissions

The extension requests the minimal permissions required for the features you enable. Examples:

  • tabs — to observe tab lifecycle events (open, update, close) so sessions can be recorded.
  • storage — to persist sessions and settings locally.
  • scripting and host permissions — to inject a small scraper into pages only when you enable automatic scraping; these permissions may be requested at install or optionally at runtime depending on settings.
  • sessions — to retrieve recently closed tab info when a tab close event occurs.

No Default Analytics

The extension does not include analytics or telemetry by default. Any telemetry or remote logging must be explicitly enabled and will be documented in this policy and the repository.

Your Choices

  • You can disable scraping or trackAllSites in settings to limit data collection.
  • Delete stored data via the extension UI or by uninstalling the extension (uninstall clears extension storage).
  • Do not enter API keys unless you trust the provider; consider hosting your own proxy if you prefer full control.

Children

The extension is not intended for children under 13 and does not knowingly collect personal information from children.

Changes

We may update this policy. The "Last updated" date will reflect changes. Major changes will be noted in the project's changelog.
